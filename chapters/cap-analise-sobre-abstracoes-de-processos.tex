% Programming Model
% Um programa rodando possui o seu próprio execution model que é definido pela linguagem. Em várias circunstâncias o programador deseja acessar um execution model diferente do seu para utilizar recursos que extrapolam aqueles definidos pela linguagem. O programming model é o mecanismo que permite acessar diferentes execution model, por sua vez, o programming model é implementado por meio de uma API. Por exemplo, um programa escrito em C, que cria threads, roda no GNU/Linux e no Windows, tem o seu próprio execution model na qual utiliza um programming model para utilizar o paralelismo, tudo isto é feito por meio de uma API construída sobre o programming model de threads (pthreads or WindowsThreads).
